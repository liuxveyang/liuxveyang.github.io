Codeforces Round #186 (Div. 2) C. Ilya and Matrix

May 31, 2013
CodeForces

题目链接：http://codeforces.com/problemset/problem/313/C 题目大意： 　　还是看原题吧，意思不难懂。 题目思路： 　　开始一直没有想法，后来听一然酱讲了才好不容易搞懂，赶脚好神奇。 　　构造的思路应该算贪心，先把前4大的数字排列到最先划分的4个方格里面，然后再依次把后面比较大的数字依次填充到这四个方格里空着的小格子里面，直到所有的格子填满为止，为什么这个思路是正确的呢？还是模拟一下4*4的格子就差不多理解了吧。 　　但是关键是怎么算出递归得到的结果？一然酱的思路很巧妙，先把数字逆序排列，再把前序和算出来，目的是方便后面的求和，然后从递归的最底层开始考虑。最底层肯定是所有的数字都包括，也就是前N个数字的和，再往上走，就是前N/4个数字的和，直到N<1为止。碉堡了……  


#include <iostream>
#include <algorithm>
#include <cstdio>
#include <cstdlib>
#include <cstring>
using namespace std;
#define LL long long int
const int MAX = 1000000*2+10;
LL a[MAX], ans, n;
bool cmp2(LL a, LL b) {
  return a > b;
}
int main(void) {
  while (~scanf("%I64d", &n)) {
    int i;
    for (i = 0; i < n; ++i) scanf("%I64d", a+i);
    sort(a, a+n, cmp2);
    for (i = 1; i < n; ++i) a[i] += a[i-1];
    ans = 0;
    while (n >= 1) {
      ans += a[n-1];
      n = n >> 2;
    }
    printf("%I64d\n", ans);
  }
  return 0 ;
}

  然后就是注意一下类型用LL…… 这回跟一然酱去网吧做CF，感触颇深，看人家神牛怎么做比赛的，给跪……谢谢一然酱~
